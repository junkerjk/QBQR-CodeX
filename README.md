# 快借快还（微信小程序）

企业内部设备借还管理小程序，支持预约申请、审批、扫码出库、扫码入库、部分归还、追加领用与操作日志留痕。

## 目录结构

```text
miniprogram/                 小程序前端代码
  app.js                     小程序入口
  app.json                   页面与 Tab 配置
  pages/
    login/                   登录绑定
    reserve/                 借用预约
    inventory/               出入库管理中心（库管）
    approval/                订单详情/审批
    outbound/                扫码出库与追加领用
    inbound/                 扫码入库与部分归还
    mine/                    个人中心
    checkout/                预留页面（占位）
    my-reserves/             预留页面（占位）
  utils/order.js             订单与资产通用工具函数

cloudfunctions/              云函数
  adminApi/                  管理端核心接口
  syncUser/                  用户绑定/解绑/静默校验
  login/                     获取微信上下文（保留）
  exportExcel/               预留（占位）
  generateQRCode/            预留（占位）

project.config.json          微信开发者工具项目配置
project.private.config.json  本地私有配置
```

## 运行方式

1. 用微信开发者工具打开项目根目录。
2. 确认 `project.config.json` 的 `miniprogramRoot` 与 `cloudfunctionRoot` 正确。
3. 在微信开发者工具中上传并部署所需云函数（至少 `adminApi`、`syncUser`）。
4. 保证 `miniprogram/app.js` 中云环境 ID 与你当前云开发环境一致。

## 核心流程

1. 用户在 `login` 页面完成姓名+手机号绑定。
2. 用户在 `reserve` 提交借用申请。
3. 库管在 `inventory` 查看待审批并进入 `approval` 同意/拒绝。
4. 审批通过后，库管扫码进入 `outbound` 放行设备（支持追加领用）。
5. 归还时扫码进入 `inbound`，支持部分归还与全部归还。

## 数据集合（约定）

- `users`: 员工信息与绑定状态。
- `orders`: 借用订单主表（审批、出入库、归还状态）。
- `assets`: 资产库存。
- `operation_logs`: 操作日志。

## 注意事项

- 页面 `checkout`、`my-reserves` 当前为占位页面，已保留路由但不参与主业务流程。
- `cloudfunctions/exportExcel`、`cloudfunctions/generateQRCode` 目前为占位实现。
- 当前代码已做多轮等价重构与清理，目标是不改变原有业务行为。
